## 1. 项目概述

使用 Flutter 重写 MarkText 开源 Markdown 编辑器，目标平台 Windows/macOS/Linux。核心改进：多语言支持（12种）、轻量化秒启动、配置持久化、新增双栏编辑模式。遵守原项目 MIT 协议。

## 2. 技术决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 编辑器实现 | 自定义（TextField + 自定义渲染） | 最大控制力，最轻量 |
| 状态管理 | Riverpod | 现代、类型安全、可测试 |
| 配置存储 | JSON 文件直接读写（dart:io） | 零依赖、人类可读、易备份 |
| README 多语言 | 单文件 + docs/i18n/ 语言链接 | 根目录整洁 |
| 开源协议 | MIT License | 与原项目一致 |

## 5. 编辑器设计

### 5.1 源代码模式
- 基于 `TextField` + `TextEditingController`
- 自定义 `TextSpan` 实现 Markdown 语法高亮
- 左侧行号栏，固定宽度，动态渲染
- 支持 Tab 缩进、自动补全括号/引号

### 5.2 视图模式
- 自研 Markdown 渲染器：Markdown 文本 → AST → Flutter Widget 树
- 支持元素：标题、段落、粗体/斜体、列表、代码块、表格、图片、链接、任务列表、分隔线
- 点击元素进入行内编辑，修改后实时重新渲染该块
- 虚拟滚动：只构建可见区域 Widget

### 5.3 双栏模式（新增）
- `Row` 布局：左侧源代码 + 可拖动分隔条 + 右侧只读预览
- 分隔条：`GestureDetector` 处理拖动，比例范围 20:80 ~ 80:20，默认 50:50
- 同步策略：
  - 编辑触发 300ms 防抖 → 增量解析变化段落 → 右侧局部刷新
  - 滚动同步：按源代码行号比例映射到预览位置
- 三种模式共享同一文本数据源，切换零开销

### 5.4 Markdown 解析器
自研轻量解析器，支持 CommonMark + GFM 子集：
- 块级：标题、段落、列表（有序/无序/任务）、代码块、引用、表格、分隔线
- 行内：粗体、斜体、删除线、行内代码、链接、图片
- 增量解析：仅重新解析变化的文本块，非全量解析

## 6. 国际化设计

- 使用 `flutter_localizations` + `intl` + `.arb` 文件
- 12 种语言：en-US, zh-CN, ja-JP, ko-KR, de-DE, fr-FR, it-IT, ru-RU, es-ES, pt-PT, ar-SA, pt-BR
- Riverpod `localeProvider` 管理当前语言
- 首次启动读取 `Platform.localeName` 自动匹配
- 阿拉伯文通过 `Directionality` Widget 支持 RTL
- 语言切换即时生效，无需重启

## 7. 配置持久化设计

- 单一 `config.json` 文件，路径：`{平台应用数据目录}/marktext-plus/config.json`
- `ConfigService` 负责读写，启动时一次性加载，变更时异步写入
- 配置模型：Dart class + `toJson()`/`fromJson()`，无代码生成
- 持久化内容：
  - 视图：侧边栏、标签栏、编辑模式、分隔比例
  - 编辑器：字体、字号、行高、自动保存
  - 主题：亮色/暗色、主题名称
  - 语言：locale 代码
  - 窗口：尺寸、位置、是否最大化
  - Markdown：列表标记、Tab 大小、是否启用 HTML
- 配置损坏时自动回退默认值

| 主题 | 类型 | 背景 | 前景 |
|------|------|------|------|
| Cadmium Light | 亮色 | #FFFFFF | #333333 |
| One Dark | 暗色 | #282C34 | #ABB2BF |
| Material Dark | 暗色 | #212121 | #FFFFFF |

支持跟随系统亮色/暗色自动切换。

## 9. 文件管理设计

- 多标签页：`TabInfo` 模型记录文件路径、内容、修改状态、光标位置、滚动偏移
- 文件树：`FileNode` 递归模型，支持展开/折叠，展开状态持久化
- 文件操作：新建、打开、保存、另存为、关闭（未保存提示）
- 最近文件列表：持久化最近 20 个文件路径
