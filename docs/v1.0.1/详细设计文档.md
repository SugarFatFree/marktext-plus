## 2. 核心模块设计

### 2.1 国际化模块 (i18n)

#### 技术方案
使用 `flutter_localizations` + `intl` 包实现国际化。

#### 架构设计

```dart
// locale_service.dart
class LocaleService {
  static const supportedLocales = [
    Locale('en', 'US'),  // 英文
    Locale('zh', 'CN'),  // 中文简体
    Locale('ja', 'JP'),  // 日文
    Locale('ko', 'KR'),  // 韩文
    Locale('de', 'DE'),  // 德文
    Locale('fr', 'FR'),  // 法文
    Locale('it', 'IT'),  // 意大利文
    Locale('ru', 'RU'),  // 俄文
    Locale('es', 'ES'),  // 西班牙文
    Locale('pt', 'PT'),  // 葡萄牙文
    Locale('ar', 'SA'),  // 阿拉伯文
    Locale('pt', 'BR'),  // 巴西葡萄牙文
  ];
}
```

#### 翻译文件结构
每种语言一个 `.arb` 文件，使用 Flutter 官方 `intl` 方案：

```json
// app_en.arb
{
  "@@locale": "en",
  "appTitle": "MarkText Plus",
  "menuFile": "File",
  "menuEdit": "Edit",
  "menuView": "View",
  "menuHelp": "Help",
  "viewSourceCode": "Source Code Mode",
  "viewPreview": "Preview Mode",
  "viewSplit": "Split View Mode",
  "settingsLanguage": "Language",
  "settingsTheme": "Theme",
  "sidebarShow": "Show Sidebar",
  "sidebarHide": "Hide Sidebar"
}
```

#### RTL 支持
阿拉伯文需要 RTL 布局，通过 `Directionality` Widget 实现：

```dart
Directionality(
  textDirection: isRtl ? TextDirection.rtl : TextDirection.ltr,
  child: child,
)
```

### 2.2 配置持久化模块

#### 技术方案
使用 `shared_preferences` 存储简单配置，`hive` 存储复杂结构化数据。

#### 配置模型

```dart
class AppConfig {
  // 视图配置
  bool sideBarVisible;
  bool tabBarVisible;
  EditMode editMode; // source, preview, split
  double splitRatio; // 双栏模式分割比例

  // 编辑器配置
  String fontFamily;
  double fontSize;
  double lineHeight;
  bool autoSave;
  int autoSaveDelay;

  // 主题配置
  ThemeMode themeMode;
  String themeName;

  // 语言配置
  String locale;

  // Markdown 配置
  String bulletListMarker;
  int tabSize;
  bool enableHtml;

  // 窗口配置
  double windowWidth;
  double windowHeight;
  double windowX;
  double windowY;
  bool isMaximized;
}
```

#### 持久化流程

```
用户修改配置 → Provider 更新状态 → UI 立即响应
                                  → ConfigService 异步写入磁盘
                                  
应用启动 → ConfigService 读取磁盘 → Provider 初始化状态 → UI 渲染
```

#### 存储位置
- Windows: `%APPDATA%/marktext-plus/config.json`
- macOS: `~/Library/Application Support/marktext-plus/config.json`
- Linux: `~/.config/marktext-plus/config.json`

#### 双栏模式实现

```dart
class SplitEditor extends StatefulWidget {
  // 核心组件结构
  Widget build(BuildContext context) {
    return Row(
      children: [
        // 左侧：源代码编辑器
        Expanded(
          flex: (splitRatio * 100).toInt(),
          child: SourceEditor(
            controller: _sourceController,
            onChanged: _onSourceChanged,
            onScroll: _syncScroll,
          ),
        ),
        // 中间：可拖动分隔条
        GestureDetector(
          onHorizontalDragUpdate: _onDragUpdate,
          child: Container(width: 4, color: dividerColor),
        ),
        // 右侧：Markdown 预览
        Expanded(
          flex: ((1 - splitRatio) * 100).toInt(),
          child: MarkdownPreview(
            data: _markdownContent,
            scrollController: _previewScrollController,
          ),
        ),
      ],
    );
  }
}
```

#### 实时同步策略

```
源代码编辑器输入
    │
    ▼
防抖处理 (300ms)
    │
    ▼
Markdown 解析 (增量)
    │
    ▼
Diff 计算 (仅变化部分)
    │
    ▼
预览视图更新 (局部刷新)
```

#### 滚动同步算法

```dart
void _syncScroll(double sourceScrollOffset) {
  // 1. 计算源代码当前可见行范围
  final visibleLineRange = _getVisibleLineRange(sourceScrollOffset);

  // 2. 通过行号映射表找到预览中对应的位置
  final previewOffset = _lineToPreviewOffset[visibleLineRange.start];

  // 3. 平滑滚动预览视图到对应位置
  _previewScrollController.animateTo(
    previewOffset,
    duration: Duration(milliseconds: 100),
    curve: Curves.easeOut,
  );
}
```

## 3. 状态管理设计

### 3.1 Provider 架构

```
ChangeNotifierProvider<SettingsProvider>    // 全局设置
ChangeNotifierProvider<ThemeProvider>       // 主题管理
ChangeNotifierProvider<LocaleProvider>      // 语言管理
ChangeNotifierProvider<TabProvider>         // 标签页管理
ChangeNotifierProvider<EditorProvider>      // 编辑器状态
ChangeNotifierProvider<FileProvider>        // 文件管理
```

### 3.2 状态流转

```
用户操作 → Provider.method() → notifyListeners()
                             → Service 层处理（文件IO、配置保存等）
                             → UI 自动重建
```

## 9. 性能优化策略

### 9.1 启动优化
- 延迟初始化非核心服务（导出、拼写检查）
- 预编译 Markdown 解析器
- 使用 `flutter build` release 模式优化

### 9.2 编辑器优化
- 虚拟滚动：仅渲染可见区域
- 增量解析：仅解析变化的文本块
- 防抖渲染：300ms 防抖避免频繁重绘

### 9.3 内存优化
- 大文件分块加载
- 图片懒加载
- 关闭标签页时释放资源

## 10. 错误处理策略

### 10.1 文件操作
- 文件不存在：提示用户，提供创建选项
- 权限不足：提示用户检查文件权限
- 编码错误：尝试自动检测编码，失败时提示用户选择

### 10.2 配置损坏
- 配置文件损坏：自动恢复为默认配置
- 版本不兼容：自动迁移配置格式
