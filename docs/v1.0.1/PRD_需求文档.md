## 1. 项目概述

### 1.1 项目背景
MarkText 是一款基于 Electron + Vue.js 的开源 Markdown 编辑器，功能强大但存在启动速度慢、资源占用高的问题。本项目使用 Flutter 重写 MarkText，在保留核心功能的基础上，实现轻量化、高性能的目标，并新增多语言支持、配置持久化、双栏编辑模式等功能。

### 1.2 项目目标
- **性能优化**: 实现秒级启动，降低内存和 CPU 占用
- **国际化**: 支持 12+ 种语言，覆盖全球主要市场
- **用户体验**: 配置持久化，减少重复操作
- **功能增强**: 新增源代码&视图双栏编辑模式
- **开源合规**: 遵守 MIT 协议，保持开源精神

### 1.3 目标用户
- 技术写作者、开发者、学生、研究人员
- 需要高性能 Markdown 编辑器的用户
- 多语言环境下的国际用户

## 2. 核心需求

### 2.1 多语言支持 (P0)

#### 需求描述
原版 MarkText 仅支持英文界面（虽然文档有多语言翻译），需要实现完整的多语言切换功能。

#### 支持语言列表
| 语言 | 语言代码 | 优先级 |
|------|---------|--------|
| 中文（简体） | zh-CN | P0 |
| 英文 | en-US | P0 |
| 日文 | ja-JP | P0 |
| 韩文 | ko-KR | P0 |
| 德文 | de-DE | P1 |
| 法文 | fr-FR | P1 |
| 意大利文 | it-IT | P1 |
| 俄文 | ru-RU | P1 |
| 西班牙文 | es-ES | P1 |
| 葡萄牙文 | pt-PT | P1 |
| 阿拉伯文 | ar-SA | P2 |
| 葡萄牙文（巴西） | pt-BR | P1 |

#### 功能要点
- 应用内语言切换（设置页面）
- 首次启动根据系统语言自动选择
- 语言切换后立即生效，无需重启
- 所有 UI 文本、菜单、提示信息均需翻译

#### 验收标准

- [ ] 语言切换功能正常，无遗漏文本

- [ ] 系统语言自动识别准确

### 2.2 轻量化/秒启动 (P0)

#### 需求描述
原版 MarkText 基于 Electron，启动时间约 3-5 秒，内存占用 200MB+。Flutter 版本需要实现秒级启动（<1s），内存占用控制在 100MB 以内。

#### 性能指标
| 指标 | 原版 (Electron) | 目标 (Flutter) |
|------|----------------|---------------|
| 冷启动时间 | 3-5s | <1s |
| 热启动时间 | 1-2s | <0.3s |
| 空闲内存占用 | 200-300MB | <100MB |
| 打开大文件 (10MB) | 2-3s | <1s |
| 安装包大小 | 100-150MB | <50MB |

#### 优化策略
- 使用 Flutter 原生渲染，避免 Chromium 内核开销
- 延迟加载非核心功能模块
- 优化资源加载（图片、字体、主题）
- 使用高效的 Markdown 解析库
- 实现虚拟滚动，处理大文件

#### 验收标准
- [ ] 冷启动时间 <1s（在中等配置电脑上）
- [ ] 空闲内存占用 <100MB
- [ ] 打开 10MB Markdown 文件 <1s
- [ ] 安装包大小 <50MB

### 2.3 配置持久化 (P0)

#### 需求描述
原版 MarkText 部分配置（如侧边栏显示状态）不会持久化，每次启动需要重新设置。需要实现完整的配置持久化系统。

#### 需要持久化的配置
**视图配置**
- 侧边栏显示/隐藏状态
- 标签栏显示/隐藏状态
- 文件树展开状态
- 窗口大小和位置
- 编辑模式（源代码/视图/双栏）

**编辑器配置**
- 主题（亮色/暗色/自定义）
- 字体、字号、行高
- 自动保存间隔
- 拼写检查开关
- 代码块语法高亮主题

**Markdown 配置**
- 列表标记样式
- 标题样式偏好
- Tab 大小
- 是否启用 HTML
- 是否启用 GitLab 兼容性

**其他配置**
- 语言设置
- 图片插入行为
- 搜索排除规则
- 快捷键自定义

#### 技术方案
- 使用 `shared_preferences` 或 `hive` 存储配置
- JSON 格式存储复杂配置
- 配置变更立即保存
- 提供配置导入/导出功能

#### 验收标准
- [ ] 所有配置项均可持久化
- [ ] 重启应用后配置保持不变
- [ ] 配置变更响应及时（<100ms）
- [ ] 支持配置重置为默认值

### 2.4 源代码&视图双栏模式 (P0)

#### 需求描述
原版 MarkText 支持两种模式：
1. **源代码模式**: 纯文本编辑（CodeMirror）
2. **视图模式**: WYSIWYG 编辑（Muya 编辑器）

新增第三种模式：
3. **双栏模式**: 左侧源代码，右侧实时预览

#### 功能要点
**布局设计**
- 窗口水平分割为左右两栏
- 左侧：源代码编辑器（语法高亮、行号）
- 右侧：实时渲染的 Markdown 视图
- 中间：可拖动的分隔条，调整左右比例
- 默认比例：50:50，可调整范围 20:80 ~ 80:20

**实时同步**
- 左侧编辑时，右侧实时更新（防抖 300ms）
- 滚动同步：左侧滚动，右侧对应位置同步滚动
- 光标位置映射：左侧光标行对应右侧渲染块高亮

**性能优化**
- 增量渲染：仅更新变化的部分
- 虚拟滚动：处理超长文档
- 防抖/节流：避免频繁渲染

**模式切换**
- 菜单栏：视图 → 编辑模式 → 源代码/视图/双栏
- 快捷键：Ctrl+1（源代码）、Ctrl+2（视图）、Ctrl+3（双栏）
- 工具栏按钮：快速切换

#### 技术方案
- 左侧编辑器：使用 `flutter_code_editor` 或 `re_editor`
- 右侧渲染：使用 `flutter_markdown` 或自定义渲染器
- 滚动同步：监听 ScrollController，计算位置映射
- 分隔条：使用 `split_view` 或自定义 Widget

#### 验收标准
- [ ] 双栏模式布局正确，分隔条可拖动
- [ ] 左侧编辑，右侧实时更新（延迟 <500ms）
- [ ] 滚动同步准确（误差 <5%）
- [ ] 大文件（10MB）下双栏模式流畅（FPS >30）
- [ ] 模式切换无卡顿

## 7. 风险与挑战

### 7.1 技术风险
- **Markdown 渲染性能**: Flutter 生态中缺少高性能的 Markdown 渲染库
  - **缓解措施**: 自研渲染引擎或优化现有库
- **双栏同步精度**: 滚动同步和位置映射可能不准确
  - **缓解措施**: 建立精确的行号-位置映射算法

### 7.2 资源风险
- **翻译质量**: 12 种语言的翻译工作量大
  - **缓解措施**: 优先完成核心语言（中英日韩），其他语言使用机器翻译+人工校对

### 7.3 兼容性风险
- **跨平台差异**: Windows/macOS/Linux 行为不一致
  - **缓解措施**: 早期在三个平台上测试，使用 Flutter 平台适配 API
